## 1. 30초 요약 (엘리베이터 피치)

> **PawTrace는 강아지 입양 투명성 플랫폼입니다.**
> 사용자가 보호소와 강아지의 이력을 지도·타임라인으로 확인할 수 있게 해서, 신뢰할 수 있는 입양 문화를 돕습니다.
>
> 기술적으로는 **FastAPI 백엔드를 Docker로 컨테이너화**해서, **Terraform(IaC)으로 정의한 AWS 인프라(ECS Fargate)** 위에 올리고,
> **GitHub Actions로 앱 배포와 인프라 배포를 모두 자동화(GitOps)** 했습니다.
> 보안은 **OIDC 키리스 인증 + DevSecOps 스캔(Trivy·SonarQube·GitGuardian 등)** 으로 처리합니다.

이 한 문단을 말할 수 있으면, 아래는 "그걸 어떻게/왜 했는지"의 디테일입니다.

## 2. 큰 그림: 전체 아키텍처

```
        👤 사용자 (브라우저)
         │
         │  ① 화면 요청 (HTML/CSS/JS)
         ▼
   ┌─────────────────┐
   │  프론트엔드       │   프로토타입(HTML) → 추후 Next.js
   │  (정적 파일)      │   * api.js 가 백엔드 API 호출
   └─────────────────┘
         │
         │  ② REST API 호출 (예: GET /api/v1/dogs/today)
         ▼
   ┌──────────────────────────────────────────────────────┐
   │                    AWS 클라우드                          │
   │                                                        │
   │   [ ALB ]  ──③──▶  [ ECS Fargate: FastAPI 컨테이너 ]    │
   │   부하분산           │  비즈니스 로직                      │
   │                     ├──④──▶ [ RDS PostgreSQL ] 데이터    │
   │                     ├──④──▶ [ ElastiCache Redis ] 캐시   │
   │                     ├──④──▶ [ S3 ] 신고 이미지            │
   │                     └──④──▶ [ Bedrock ] AI 요약·분류      │
   └──────────────────────────────────────────────────────┘
         ▲
         │  ⑤ 코드를 main 에 머지하면 자동 배포
   ┌─────────────────┐
   │ GitHub Actions   │   deploy.yml (앱) · infra.yml (인프라)
   └─────────────────┘
```

**한 문장씩 읽기:**
- **사용자**는 브라우저로 화면을 받고(①), 그 화면의 JavaScript가 백엔드에 데이터를 요청합니다(②).
- 요청은 먼저 **ALB(로드밸런서)** 에 도착하고(③), ALB가 **ECS Fargate에서 돌아가는 FastAPI 컨테이너**로 전달합니다.
- FastAPI는 필요에 따라 **DB(RDS)·캐시(Redis)·파일(S3)·AI(Bedrock)** 를 사용해 응답을 만듭니다(④).
- 개발자가 코드를 고쳐 **main에 머지하면 GitHub Actions가 자동으로 빌드·배포**합니다(⑤).

> 📌 상세 다이어그램(Mermaid PNG)은 [`docs/architecture/aws-infrastructure.md`](architecture/aws-infrastructure.md)에 있습니다.

## 3. 요청 하나의 여정 (Request Lifecycle)

**"사용자가 홈에서 오늘의 친구(봄이)를 볼 때, 실제로 무슨 일이 일어나는가?"**
이걸 설명할 수 있으면 시스템을 이해한 것입니다.

```
1. 브라우저가 index.html 을 엽니다.
2. index.html 안의 api.js 가 실행됩니다.
3. api.js 가 fetch("http://<ALB주소>/api/v1/dogs/today") 호출.
        │
4.      └─▶ 인터넷 → AWS의 ALB(로드밸런서)에 도착.
5.          ALB는 "건강한" ECS 컨테이너 하나를 골라 요청 전달.
6.          FastAPI 라우터(dogs.py)가 /today 요청을 받음.
7.          → 서비스(dog_service) → 리포지토리(dog_repository) → 데이터 조회.
                (지금은 seed 데이터, 실제 운영은 RDS PostgreSQL)
8.          FastAPI가 JSON 응답 생성: {"name":"봄이", "shelter_name":"햇살보호소", ...}
        ┌─────────────────────────────────────────────────┘
9.      ▼ 응답이 ALB를 거쳐 다시 브라우저로.
10. api.js 가 JSON을 받아 화면의 이름/사진/스토리를 교체.
11. 우상단 배지가 "● 실시간 API" 로 바뀜. (실패하면 "○ 데모 데이터")
```

이 흐름에서 등장하는 모든 조각(ALB, ECS, FastAPI 레이어, CORS, 폴백)을 아래에서 하나씩 설명합니다.

## 4. 레이어별 상세 + 용어 풀이

### 4-1. 프론트엔드 (사용자가 보는 화면)

- **무엇:** 지금은 빌드가 필요 없는 **정적 HTML/CSS/JS 프로토타입**(`docs/prototype/`). 추후 **Next.js**로 정식 구현 예정.
- **핵심 파일 `api.js`** — 백엔드 연결을 담당. 우리가 쓴 패턴 2가지:
  - **점진적 향상(Progressive Enhancement):** HTML에 "데모 데이터"가 먼저 박혀 있어, **백엔드가 꺼져 있어도 화면이 안 깨짐**. 백엔드가 살아 있으면 JS가 실제 데이터로 교체.
  - **연결 상태 배지:** `● 실시간 API` / `○ 데모 데이터` 로 현재 상태를 눈으로 확인.
- 💬 **쉬운 비유:** 식당 메뉴판(HTML)은 항상 보이고, 주방(백엔드)이 열려 있으면 "오늘의 특선"을 실시간으로 덧붙여 주는 것.

> **CORS (Cross-Origin Resource Sharing)** 란?
> 브라우저 보안 규칙. "A 주소의 웹페이지가 B 주소의 API를 호출해도 되는가?"를 B(백엔드)가 허락해야 함.
> 우리 백엔드는 `CORS_ORIGINS` 설정으로 허용 주소를 지정합니다. (실무에서 자주 만나는 에러 1순위!)

### 4-2. 백엔드 (FastAPI · Clean Architecture)

- **FastAPI:** 파이썬 웹 프레임워크. 빠르고, 자동으로 API 문서(`/docs`)를 만들어 줌.
- **Clean Architecture(계층 분리):** 코드를 역할별로 나눠서 유지보수를 쉽게 함.

  ```
  api/      ← 요청을 받는 입구 (URL ↔ 함수 연결)         "웨이터"
   └ routes/dogs.py, shelters.py, reports.py
  services/ ← 비즈니스 로직 (규칙, 가공)                  "주방장"
  repositories/ ← 데이터 읽기/쓰기 (지금은 seed, 추후 DB)  "창고지기"
  schemas/  ← 요청/응답의 형태 정의 (Pydantic)            "주문서 양식"
  domain/   ← 핵심 개념(enum 등). 점수/랭킹 없음(윤리 원칙)  "사전"
  integrations/ ← 외부 연동(AI, S3, 공공데이터) 격리        "외부 거래처"
  ```
  💬 **왜 나누나?** 나중에 "seed 데이터 → 진짜 DB"로 바꿀 때 **repositories만 고치면** 되고, 웨이터(api)·주방장(services)은 안 건드려도 됨.

- **seed 데이터:** MVP를 빨리 보여주려고 DB 없이 코드 안에 넣어둔 샘플(봄이, 햇살보호소). `repositories/seed.py`.

### 4-3. 컨테이너 (Docker · ECR)

- **Docker:** 앱 + 실행환경(파이썬, 라이브러리)을 **하나의 상자(이미지)** 로 묶는 기술.
  "내 컴퓨터에선 됐는데요?" 문제를 없앰 — 어디서 돌려도 똑같이 동작.
- **이미지 vs 컨테이너:** 이미지 = 붕어빵 **틀**, 컨테이너 = 그 틀로 찍어낸 **붕어빵**(실행 중인 것).
- **멀티스테이지 빌드:** 빌드용 도구는 버리고 **실행에 필요한 것만** 최종 이미지에 담아 가볍고 안전하게.
- **ECR (Elastic Container Registry):** AWS의 **도커 이미지 창고**. 빌드한 이미지를 여기 올리면 ECS가 가져다 실행.

### 4-4. AWS 인프라 (실제 서버들)

| 용어 | 한 줄 설명 | 쉬운 비유 |
|---|---|---|
| **VPC** | 내 전용 가상 네트워크(울타리) | 우리 회사 전용 건물 |
| **Subnet** | VPC를 나눈 구역(공개/비공개) | 건물 안의 층/방 |
| **Public Subnet** | 인터넷과 직접 연결된 구역 | 1층 로비(외부인 출입) |
| **Private Subnet** | 인터넷에서 직접 못 들어오는 구역 | 내부 사무실(직원만) |
| **ALB** (Application Load Balancer) | 들어오는 요청을 건강한 서버로 분배 | 안내 데스크 |
| **ECS Fargate** | 서버 관리 없이 컨테이너를 실행 | 서버 없는 주방(요리만 신경) |
| **RDS** | AWS가 관리해주는 데이터베이스(PostgreSQL) | 관리형 창고 |
| **ElastiCache (Redis)** | 빠른 임시 저장소(캐시) | 자주 쓰는 물건 책상 위에 |
| **S3** | 파일 저장소(이미지 등) | 무한 사물함 |
| **Secrets Manager** | 비밀번호/키를 안전히 보관 | 금고 |
| **NAT Gateway** | 내부 서버가 인터넷으로 **나갈** 때 통로 | 직원 전용 출구 |
| **Security Group** | 어떤 통신을 허용할지 정하는 방화벽 | 출입 통제 규칙 |

💬 **왜 이렇게 나누나(공개/비공개)?** DB(RDS)는 **Private Subnet**에 둬서 인터넷에서 직접 접근 불가 →
오직 ALB→ECS를 거친 요청만 닿을 수 있게 함. **보안의 기본 원칙(최소 노출).**

### 4-5. IaC (Terraform) — "인프라를 코드로"

- **IaC (Infrastructure as Code):** 위의 VPC·ALB·RDS 같은 걸 **콘솔에서 클릭으로** 만들지 않고
  **코드(`.tf` 파일)로** 정의해서 명령 한 번에 만들고 지움.
- **왜 좋은가:**
  - 똑같은 인프라를 **5분이면 재현** (비용 아끼려 지웠다가 다시 올리기 쉬움).
  - 변경 이력이 **Git에 남음** (누가 무엇을 왜 바꿨는지).
  - 사람 실수(클릭 누락)를 줄임.
- **우리 구조 — 폴더 2개로 분리:**

  | 폴더 | 무엇을 만드나 | 실행 빈도 | 비용 |
  |---|---|---|---|
  | `infra/terraform/bootstrap/` | IAM 역할·ECR·상태저장소(S3) | **딱 1회** | 거의 $0 |
  | `infra/terraform/` (메인) | VPC·ALB·ECS·RDS·Redis·S3 | 필요할 때마다 | 시간당 과금 💰 |

- **terraform state(상태 파일):** Terraform이 "내가 뭘 만들었는지" 기억하는 장부.
  - 처음엔 로컬 파일이었는데, **CI도 공유**할 수 있게 **S3 + DynamoDB(잠금)** 으로 옮김(원격 상태).
  - 💬 우리가 겪은 버그: 상태 파일이 다른 폴더에 있어서 "이미 존재한다" 에러가 났고, 진짜 상태를 복사해 해결함. → **"상태 파일이 진실의 원천"** 임을 체험.
- **주요 명령:**
  - `terraform plan` — "이렇게 바꿀 건데 괜찮아?" **미리보기**(실제 변경 X)
  - `terraform apply` — 실제로 만들기/바꾸기
  - `terraform destroy` — 다 지우기(과금 중단)

## 5. CI/CD: 두 개의 자동화 파이프라인

> **CI/CD** = 코드를 합칠 때마다 **자동으로 테스트·빌드·배포**하는 것.
> CI(Continuous Integration, 지속적 통합) + CD(Continuous Delivery/Deployment, 지속적 배포).

우리는 **GitHub Actions**로 파이프라인을 2개 만들었습니다.

### 5-1. `deploy.yml` — 앱(코드) 배포
```
main 에 코드 푸시
  → Docker 이미지 빌드
  → 🛡️ Trivy 보안 스캔 (취약점 있으면 중단)
  → 📦 SBOM 생성 (어떤 부품을 썼는지 목록)
  → 🔑 OIDC로 AWS 인증 (액세스키 없이!)
  → ECR에 이미지 push
  → ECS 서비스 롤링 업데이트 (무중단 교체)
```

### 5-2. `infra.yml` — 인프라 배포 (GitOps)
```
PR에서 infra/terraform/** 수정
  → terraform plan 실행
  → 📝 plan 결과를 PR에 코멘트로 게시  ← 사람이 리뷰
  → 승인 후 main에 머지
  → terraform apply 자동 실행
```
💬 이게 **GitOps**: "Git에 머지하는 것 = 배포". 인프라 변경도 코드 리뷰를 거침.

### 5-3. 보안 도구 (DevSecOps) — "파이프라인에 보안을 심다"

| 도구 | 역할 | 한 줄 |
|---|---|---|
| **Trivy** | 컨테이너 취약점 스캔 | 도커 이미지에 알려진 보안 구멍 있나? |
| **GitGuardian** | 시크릿 유출 탐지 | 실수로 비밀번호/키를 커밋했나? |
| **SonarQube** | 코드 품질·보안 | 버그·코드 냄새·취약 패턴 |
| **Hadolint** | Dockerfile 린트 | 도커 작성 모범사례 위반? |
| **Codecov** | 테스트 커버리지 | 테스트가 코드를 얼마나 덮나? |
| **Syft (SBOM)** | 부품 목록 생성 | 공급망 투명성(어떤 라이브러리?) |
| **Dependabot** | 의존성 자동 업데이트 | 오래된/취약한 패키지 알림 |

### 5-4. OIDC — "비밀번호 없는 인증"의 핵심

- **문제:** GitHub Actions가 AWS에 배포하려면 AWS 자격증명이 필요. 그런데 **액세스키를 GitHub에 저장하면** 유출 위험.
- **해결(OIDC):** GitHub가 발급한 **단기 신분증(토큰)** 을 AWS가 신뢰하게 설정. AWS가 "이 리포의 main 브랜치라면 잠깐 들어와"라고 **임시 권한**을 줌.
- 💬 **비유:** 영구 열쇠를 복사해 주는 대신, 그때그때 **유효기간 5분짜리 출입증**을 발급. 잃어버려도 곧 만료.
- **우리는 역할을 2개로 분리:**
  - `pawtrace-gha-deploy` — 앱 배포용 (ECR·ECS만)
  - `pawtrace-gha-terraform` — 인프라용 (더 넓은 권한)
  - 💬 **왜?** **직무 분리(최소 권한)** — 앱 배포가 인프라를 못 건드리게.

## 6. 우리가 만든 과정 (스토리 타임라인)

면접에서 "이 프로젝트 어떻게 진행했어요?"에 답하는 스토리입니다.

1. **기획** — 서비스 정의, 페르소나, MVP 범위, ERD, API 설계, 윤리 원칙(강아지 점수화 금지) 수립.
2. **이슈화** — GitHub Issue 71개 + 프로젝트 보드로 작업 분해.
3. **프로토타입** — 둥근/파스텔 테마의 정적 화면(홈·지도·Pet Passport) 제작.
4. **백엔드 스캐폴딩** — FastAPI를 Clean Architecture로 구성, seed 데이터로 즉시 동작, pytest 통과.
5. **컨테이너화** — Dockerfile(멀티스테이지) + docker-compose(api+DB+Redis).
6. **클라우드 전환** — Azure 계획을 **AWS로 마이그레이션**(문서·다이어그램 갱신).
7. **DevSecOps CI/CD** — Trivy·SonarQube·GitGuardian 등 보안 게이트를 파이프라인에 추가.
8. **IaC 작성** — Terraform으로 bootstrap(OIDC·ECR·상태) + 메인 인프라(VPC~RDS) 정의.
9. **실배포** — ECR에 이미지 push → `terraform apply`로 39개 리소스 생성 → **ALB 주소로 API 동작 확인**(봄이 JSON 응답).
10. **인프라 자동화** — `infra.yml`로 plan-on-PR / apply-on-merge **GitOps** 구축, 테스트 PR로 검증.
11. **프론트 연결** — 프로토타입을 실제 API에 연결(폴백 포함), 헤드리스 브라우저로 E2E 검증.
12. **비용 관리** — 데모 후 `terraform destroy`로 과금 중단, 필요 시 재생성.

💬 **막혔다가 푼 경험(면접 가산점):**
- ECR push "no basic auth credentials" → **ECR 로그인 누락**이었음(같은 터미널에서 login 후 push).
- VS Code에서 `aws` 명령 인식 안 됨 → **PATH는 프로세스 시작 시 읽힘** → VS Code 완전 재시작으로 해결.
- bootstrap "already exists" → **상태 파일이 다른 폴더**에 있어서 → 진짜 상태를 복사해 해결.

## 8. 면접 예상 질문 & 모범 답변

**Q1. 이 프로젝트를 한마디로 설명하면?**
> 강아지 입양 투명성 플랫폼입니다. 보호소·강아지 이력을 지도와 타임라인으로 보여주고, FastAPI 백엔드를 Docker로 컨테이너화해 Terraform으로 정의한 AWS ECS Fargate에 배포했습니다. 앱·인프라 배포 모두 GitHub Actions로 자동화했습니다.

**Q2. 왜 ECS Fargate를 골랐나요? EC2나 EKS가 아니라?**
> 1인 개발 MVP라 **서버 관리 부담을 줄이는 게** 우선이었습니다. Fargate는 EC2 인스턴스를 직접 관리할 필요가 없어 운영이 단순합니다. EKS(쿠버네티스)는 학습/운영 비용이 커서, 트래픽이 커지면 확장하는 단계로 남겨뒀습니다.

**Q3. 인프라를 왜 Terraform으로 했나요? 콘솔로 만들면 안 되나요?**
> 콘솔 클릭은 재현·이력 관리가 안 됩니다. Terraform은 인프라를 코드로 남겨 **5분이면 동일하게 재생성**하고, 변경을 Git으로 리뷰할 수 있습니다. 실제로 비용 절약을 위해 destroy했다가 파이프라인으로 다시 올릴 수 있게 설계했습니다.

**Q4. 보안은 어떻게 신경 썼나요?**
> 세 가지입니다. ① **OIDC 키리스 인증**으로 액세스키를 저장하지 않고, ② DB를 **Private Subnet**에 두어 인터넷 직접 접근을 막고, ③ 파이프라인에 **Trivy·GitGuardian·SonarQube** 등 DevSecOps 게이트를 넣어 취약점·시크릿 유출을 자동 차단합니다.

**Q5. CI/CD 파이프라인을 설명해보세요.**
> 두 개로 분리했습니다. `deploy.yml`은 앱 코드를 빌드·스캔·ECR push·ECS 롤링 배포하고, `infra.yml`은 인프라를 PR에서 plan으로 리뷰하고 머지하면 apply하는 GitOps 방식입니다. 역할도 앱용·인프라용으로 분리해 최소 권한을 지켰습니다.

**Q6. 가장 어려웠던 문제와 해결은?**
> 부트스트랩 재실행 때 "이미 존재한다" 에러가 났는데, 원인은 **Terraform 상태 파일이 다른 폴더**에 있어 현재 폴더가 기존 리소스를 인식하지 못한 것이었습니다. 진짜 상태 파일을 현재 폴더로 복사해 해결하며 **"상태가 진실의 원천"** 임을 체감했습니다.

**Q7. 강아지에 점수를 매기지 않는 이유는?**
> 윤리·법적 리스크 때문입니다. 특정 업체를 "불법"으로 단정하면 명예훼손 위험이 있어, **공공데이터·신고·검증 기반의 '투명성 지표'**(정보가 충분한지)로만 표현하고, 강아지 자체는 절대 평가/랭킹하지 않습니다.

## 9. 비용 & 운영 전략

- **과금 주체:** RDS · NAT Gateway · ALB · ElastiCache 는 **켜져 있는 동안 시간당 과금**(대략 시간당 $0.1~0.2).
- **거의 무료:** IAM 역할 · 빈 ECR · S3 상태 버킷 · DynamoDB(잠금).
- **전략:** 평소엔 `terraform destroy`로 내려두고, **데모/면접 직전에** `apply`로 5분 만에 올림.
  - 올리기: `cd infra/terraform && terraform apply -var="container_image=<ECR>:v1"`
  - 내리기: `terraform destroy`
- 💬 면접 어필: *"비용 관리를 위해 IaC로 필요할 때만 환경을 올립니다."*
